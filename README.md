# java-convenience-store-precourse

## 요구사항

- [ ] 상품 가격, 수량을 기반으로 최종 금액 계산(프로모션, 할인 반영)
- [ ] 구매내역, 산출 금액을 영수증으로 출력
- [ ] 영수증 출력 후 추가구매 진행 여부 선택
- [ ] IllegalArgumentException, "[ERROR]"로 에러 출력
- [ ] 재고, 프로모션 파일로 입력

- 재고 관리
  - [ ] 재고 정보를 정확히 저장, 제공
- 프로모션 할인
  - [ ] 프로모션 기간에만 할인 적용
  - [ ] N + 1 형태
  - [ ] 프로모션 재고 내에서만 적용
  - [ ] 프로모션 기간 중엔 프로모션 재고 차감, 아닐 시 일반 재고 차감
  - [ ] 프로모션 적용 가능 상품을 해당 수량보다 적게 가져온 경우 메시지 출력
  - [ ] 프로모션 재고가 부족하여 일부를 정가로 계산해야하는 경우 메시지 출력
- 멤버십 할인
  - [ ] 프로모션 적용 후 남은 금액의 30% 할인 할인 적용
  - [ ] 최대 8천원
- 영수증 출력
  - [ ] 상품 정력
  - [ ] 모든 정보 포함

## 애러

- [ ]  구매 상품, 수량이 올바르지 않은 경우 -> "올바르지 않은 형식으로 입력했습니다. 다시 입력해 주세요."
- [ ]  존재하지 않는 상품 입력 -> "존재하지 않는 상품입니다. 다시 입력해 주세요."
- [ ]  구매 수량이 재고 수량 초과 -> "재고 수량을 초과하여 구매할 수 없습니다. 다시 입력해 주세요."
- [ ]  기타 -> "잘못된 입력입니다. 다시 입력해 주세요."

## 구현할 기능

- domain
    - 창고
      - [ ] 재고 저장(일반, 프로모션)
      - [ ] 물건이 개수만큼 존재하는지 확인
      - [ ] 재고 수량 변경
    - 제품
      - [ ] 이름, 가격, 프로모션 저장
    - 고객
      - [ ] 멤버십 유무 저장
    - 영수증
      - [ ] 구매 상품 내역(상품명, 수량, 가격), 증정 상품 내역, 금액 정보(총 구매액, 행사 할인, 멤버십 할인, 최종 금액) 포함
      - [ ] 영수증 구성 요소 정렬
      - [ ] 멤버십 할인 기능
    - 프로모션
      - [ ] 이름, 구매 개수, 제공 개수
      - [ ] 시작날짜, 끝날짜
      - [ ] 특정 날짜가 프로모션 기간인지 반환
    - 구매
      - [ ] 프로모션 적용 제품, 미적용 제품, 증정 제품

- util
  - [ ] csv 파서
- controller
    - [ ]  입력 받고 서비스로 넘긴 후 반환 받은 dto 출력
    - [ ]  에러일 경우 출력 후 다시 입력받기
- service
    - [ ] 제품명, 개수 -> 구매
    - PromotionChecker
      - [ ] 프로모션 적용이 가능한지 체크
- dto
  - [ ] 재고 dto
  - [ ] 영수증 dto
